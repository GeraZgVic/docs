## Duplicate Order ID

La API de Kueski no genera error por `order_id` duplicado. El comportamiento 
depende del status del pago existente:

| Status existente | Comportamiento |
| :--- | :--- |
| `approved` / `success` | Retorna el pago existente â€” no crea uno nuevo |
| `created` / `processing` / `authorized` / `refunded` | Crea nuevo pago â€” el anterior queda como `failed` |
| `failed` / `canceled` / `denied` | Crea nuevo pago â€” el anterior se ignora |

> [!NOTE]
> En CS-Cart esto no aplica en reintentos reales â€” cada intento genera un 
> `order_id` Ăºnico basado en timestamp. El Ăºnico escenario donde aplica 
> es doble-click, cubierto por session lock.
